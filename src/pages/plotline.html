    Web App Microservice DSB-3 AWS

    Designe to confidently identify DSB 3 attackers, this profile is built with three microservices. First an attacker will identify a script containing a S3 backup URI. From that URI the attacker will derive a private key to compromise the EC2 jump box. Using that jumpbox, the attacker will exploit a FTP server in the final simuated app engine.

  • S3 Backup Bucket
    92c75190-f3ff-11ec-a661-000c2970a8e4
    This S3 bucket contians a number of backups identified by a script put on the system using endpoint protection.
  • EC2 Microservice Jump Box
    859a277c-f3ff-11ec-a661-000c2970a8e4
    A private key derived from the previous backups allow SSH access with the "ubuntu" user on the default port 22/tcp.
  • ECS Web App Engine
    b5dd020a-0652-11ed-8900-000c2970a8e4
    Using the jumbox compromise, the attacker compromises internally accessible microservices as a webapp.
